Reroll Settings

👥 Friend ID Configuration

🆔 Friend ID
Enter your Pokemon TCG Pocket Friend ID for friend battles and social features. Only required when using "Inject for Reroll" method. Leave empty if using the ids.txt API instead. This ID will be used across all instances for friend interactions.

⚡ Instance Management

📱 Instances
Number of regular reroll instances to run. Each instance handles one account simultaneously. Recommended range: 1-20 depending on your system specifications and CPU performance.
⏱️ Instance Start Delay
Delay in seconds between launching instances. Helps prevent system overload during startup. Recommended: 2-5 seconds for optimal performance. Set to 0 for simultaneous launch if your system can handle it.
📊 Columns
Number of instances per row when arranging windows. Affects the grid layout of your instance windows. Recommended: 3-6 depending on screen size and resolution. Works with Row Gap to create organized layouts.
⭐ Run Main Instance(s)
Enable Main instance(s) for primary account(s). Main instances can run different logic than reroll instances and are useful for keeping main accounts active while rerolling others. Specify number of Main instances when enabled.
🎮 Account Name
Display name for this bot configuration. Used in Discord notifications and logs to help identify different bot setups when running multiple configurations. Appears in heartbeat messages and status updates.

⏰ Timing Settings

⚡ Action Delay
Base delay between actions in milliseconds. Lower values mean faster execution but less stability. Recommended: 200-500ms for optimal balance. Increase if experiencing detection issues.
⏳ Wait Time
Wait time in seconds for various operations including loading screens and animations. Recommended: 3-10 seconds depending on device performance and connection speed.
👆 Swipe Speed
Speed of swipe animations in milliseconds. Affects pack opening and menu navigation smoothness. Recommended: 200-500ms. Lower values for faster animations, higher for more realistic movement.
🐌 Slow Motion Mode
Enable compatibility mode for slower devices or unstable connections. Increases delays for better stability. Use if experiencing frequent crashes, detection issues, or connection timeouts.

System Settings

🖥️ Display Configuration

📺 Monitor Selection
Select which monitor to display instances on. Shows all available monitors with their resolutions. All instances will arrange on the selected monitor according to your column and row gap settings.
🔍 UI Scale
UI scaling setting for MuMu instances. Must match your MuMu display settings exactly:
  • Scale125: 125% UI scaling (default) - scaleParam: 277px
  • Scale100: 100% UI scaling - scaleParam: 287px
Critical: This must match your MuMu Player scaling or clicking will be inaccurate.
📏 Row Gap
Vertical spacing between rows of instances in pixels. Recommended: 50-150 pixels. Adjust based on your screen resolution and preference. Higher values provide more space between rows.

📁 File System

📂 MuMu Folder Path
Path to MuMu Player installation directory. Default: C:\Program Files\Netease. Required for launching instances automatically through LaunchAllMumu.ahk script.

🌐 Language Settings

👁️ OCR Language
Language for Optical Character Recognition. Must match your game client language for accurate text detection and card recognition.
Supported: en, zh, es, de, fr, ja, ru, pt, ko, it, tr, pl, nl, sv, ar, uk, id, vi, th, he, cs, no, da, fi, hu, el, zh-TW
🎮 Client Language
Pokemon TCG Pocket client language. Must match your in-game language setting exactly for proper menu navigation and text recognition.
Supported: en, es, fr, de, it, pt, jp, ko, cn

🔧 Advanced System Options

🚀 Instance Launch Delay
Delay in seconds when launching MuMu instances through LaunchAllMumu.ahk. Prevents system overload during startup. Recommended: 3-10 seconds based on system performance.
📊 Auto Launch Monitor
Automatically launch Monitor.ahk with the bot. The monitor script tracks instance health, performance, and can restart failed instances. Recommended to keep enabled for stability.

🛠️ Extra Settings

🎯 Apply Role-Based Filters
Enable role-based filtering for advanced users. Provides additional filtering options for specific use cases. Leave unchecked unless you understand its purpose and have specific filtering requirements.
🐛 Debug Mode
Enable debug logging and additional information. Creates detailed logs for troubleshooting issues. Only enable when debugging problems as it may impact performance and create large log files.
🔬 Use Tesseract OCR
Use Tesseract OCR instead of built-in OCR. Requires separate Tesseract installation at the specified path. May improve text recognition accuracy in some cases but requires additional setup.
📁 Tesseract Path
Full path to tesseract.exe executable. Only visible when "Use Tesseract OCR" is enabled. Default: C:\Program Files\Tesseract-OCR\tesseract.exe. Ensure Tesseract is properly installed.
💬 Status Messages
Show status messages during operation. Displays current bot actions, progress, and important notifications. Recommended to keep enabled for monitoring bot activity and troubleshooting.

Pack Settings

⭐ God Pack Configuration

✨ Minimum 2★ Cards
Minimum 2-star cards required to keep a pack. Packs with fewer 2-star cards will be rerolled. Range: 0-5 (0 = keep all packs regardless of stars). Higher values are more selective but slower.
🌟 Minimum 2★ for Shiny Packs
Separate minimum 2-star requirement specifically for Shiny packs. Usually set lower than regular minimum since Shiny packs are rarer and more valuable even with fewer 2-star cards.

🔄 Reroll Methods

📋 Delete Method
Choose your reroll strategy:
  • 13 Pack: Traditional method - delete account after opening 13 packs
  • Inject: Advanced account injection system for experienced users
  • Inject Missions: Injection with automatic mission completion
  • Inject for Reroll: Injection with pack count thresholds and Friend ID support
1️⃣ 1 Pack Method
Open only 1 pack per account instead of multiple. Enables faster rerolls but provides fewer chances per account. Good for high-volume rerolling when targeting specific cards.
💥 Menu Delete
Delete accounts through the in-game menu instead of file deletion. Only available for non-Inject methods. Provides a cleaner deletion process and may be more stable.
⏰ Spend Hourglass
Automatically spend Hourglasses for extra pack openings. Uses in-game currency for additional opportunities. Only available for Inject methods. Increases pack opening efficiency.
📊 Sort By (Inject Methods)
Sort order for account injection when using Inject methods:
  • Oldest First: Process accounts by modification date (oldest first)
  • Newest First: Process accounts by modification date (newest first)
  • Fewest Packs First: Process accounts with fewer packs first
  • Most Packs First: Process accounts with more packs first

🔍 Card Detection & Saving

🎨 Single Full Art
Save accounts with single Full Art cards. Keeps accounts that pull any Full Art card, perfect for collection building and obtaining alternate artwork versions.
👤 Single Trainer
Save accounts with single Trainer cards. Keeps accounts that pull any Trainer card, useful for competitive deck building and obtaining supporter cards.
🌈 Single Rainbow
Save accounts with single Rainbow Rare cards. Keeps accounts that pull any Rainbow card - very rare and valuable pulls with special holographic effects.
⭐ Pseudo God Pack (Double 2★)
Save accounts with 2 or more 2-star cards in a single pack. Good middle-ground between god packs and single card saves. Provides decent value without being too restrictive.
👑 Save Crown Rares
Save accounts with Crown Rare cards. Keeps accounts that pull Crown cards - extremely rare special cards with unique artwork and premium value.
✨ Save Shiny Cards
Save accounts with Shiny cards. Keeps accounts that pull any Shiny variant - special artwork versions of existing cards with different color schemes.
🎬 Save Immersive Cards
Save accounts with Immersive cards. Keeps accounts that pull Immersive art cards - special full-screen artwork cards with unique animations and premium presentation.
🌟 Only Apply to Shiny Packs
Only apply the above card detection logic to Shiny packs. Regular packs will use standard god pack logic based on star count. Useful when hunting specific Shiny cards while maintaining efficiency on regular packs.
❌ Ignore Invalid Packs
Skip packs that can't be properly scanned due to OCR or detection errors. Helps avoid false positives and improves accuracy by excluding packs with poor image quality or recognition issues.

💗 Enable Heartbeat
Enable periodic status updates to Discord. Sends regular 'alive' messages with statistics to help monitor bot health remotely. Essential for tracking bot performance when away from computer.

Download Settings

🌐 External Data Sources

🆔 Main IDs API URL
API URL for friend ID list (ids.txt). Automatically downloads updated friend ID lists from your specified URL every 5 minutes during bot operation. Leave empty to use local ids.txt file instead. The bot will automatically delete local ids.txt when using API.
⭐ VIP IDs API URL
API URL for VIP friend ID list (vip_ids.txt). Separate list for priority friend IDs used for special account management and priority interactions. Useful for maintaining separate lists for different purposes.

📋 Friend ID Management

📄 Local vs API Priority
When API URLs are provided, they take priority over local files:
  • API Active: Downloads and uses remote lists, local files are ignored
  • API Empty: Uses local ids.txt and vip_ids.txt files if they exist
  • Friend ID Field: Only used for "Inject for Reroll" method when APIs are empty
🔄 Auto-Update Behavior
The bot automatically manages friend ID lists:
  • Startup: Downloads fresh lists before starting
  • Every 5 minutes: Updates lists during operation
  • Deprecated URLs: Warns when Friend ID field contains URLs (should use API instead)

🎪 Showcase Automation

✨ Enable Showcase
Enable showcase automation using showcase_ids.txt. Requires the showcase_ids.txt file in your bot directory for automatic showcase interactions. The bot will warn if enabled but file is missing.
📁 Showcase Files Required
For showcase functionality, you need:
  • showcase_ids.txt: List of showcase IDs to interact with (required)
  • showcase_codes.txt: Downloaded from Showcase URL if provided (optional)
Create showcase_ids.txt manually in the bot directory with one ID per line.
🎯 Showcase Likes
The bot automatically sets showcase likes to 5 when showcase is enabled. This determines how many likes to give to showcases during interactions.

⚠️ Important Notes

🔒 API Security
When using API URLs:
  • Use HTTPS: Ensure your API endpoints use secure connections
  • Access Control: Protect your API endpoints from unauthorized access
  • Rate Limiting: Bot downloads every 5 minutes, ensure your API can handle this frequency
📝 File Format
All ID files should contain:
  • One ID per line: Each friend ID on a separate line
  • Clean format: No extra spaces, commas, or special characters
  • Valid IDs only: Only include working Pokemon TCG Pocket friend IDs

Getting Started Tutorial

🏁 First Time Setup

Step 1: Initial Configuration
Navigate to Reroll Settings (Ctrl+1):
  1. Enter your Friend ID (optional if using API)
  2. Set Instances to 1 for initial testing
  3. Set Columns to 1 for single instance
  4. Configure Account Name for identification in Discord
  5. Leave Main instances disabled for now
Timing Settings:
  • Keep default Delay (250ms) and Wait Time (5s)
  • Use default Swipe Speed (300ms)
  • Don't enable Slow Motion unless on very old hardware
Step 2: System Configuration
Go to System Settings (Ctrl+2):
  1. Select your primary monitor from the dropdown
  2. Set Scale to match your MuMu settings (usually Scale125)
  3. Set Row Gap to 100 pixels (good default spacing)
  4. Verify MuMu folder path points to your installation
  5. Set OCR Language to match your game language
  6. Set Client Language to match your in-game language
  7. Keep Auto Launch Monitor enabled
Step 3: Pack Method Setup
Access Pack Settings (Ctrl+3):
  1. Set minimum 2★ cards to 2-3 for testing
  2. Start with "13 Pack" method (easiest for beginners)
  3. Select 1-2 pack types to focus on (recommend Solgaleo or Charizard)
  4. Enable "Single Full Art" for additional saves
  5. Leave other card detection disabled initially
Important: Don't enable too many detection options initially - start simple and add more as you understand the system.
Step 4: Optional Discord Setup
Go to Discord Settings (Ctrl+5) - Optional but recommended:
  1. Create a Discord webhook in your server
  2. Copy your Discord User ID (User Settings > Advanced > Developer Mode)
  3. Paste webhook URL and User ID
  4. Enable heartbeat with 30-minute intervals
  5. Set a recognizable heartbeat name
Step 5: First Launch
Time to test your setup:
  1. Click Launch All MuMu (F1) to start your instance
  2. Wait for MuMu to fully load and show the Pokemon TCG Pocket game
  3. Click Arrange Windows (F2) to position properly
  4. Click Start Bot (F3) and carefully review the confirmation
  5. Watch the first few rerolls to ensure everything works correctly
Watch for: Proper clicking, card detection, and smooth navigation.

📈 Scaling Up Your Operation

Adding More Instances
Once comfortable with single instance:
  1. Gradually increase instance count (try 2, then 3, then 5)
  2. Adjust Columns based on your screen (3-4 for 1080p, 5-6 for 1440p)
  3. Increase Row Gap if windows overlap (100-150px)
  4. Add Instance Start Delay (2-5 seconds) to prevent system overload
  5. Monitor system performance and adjust accordingly
Performance tips: Add instances slowly and test stability at each step.
